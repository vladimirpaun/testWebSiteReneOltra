Webservice de Recherche de Durées de Séjour Disponibles

Vue d'ensemble

Le webservice cpg_StayDuration permet d'interroger le système WebCamp afin d'obtenir la liste des durées de séjour disponibles pour des critères de recherche spécifiques. Ce service SOAP retourne les durées possibles (en nombre de jours) pour des hébergements disponibles selon un jour de la semaine, un mois et une année donnés.

Logique fonctionnelle

Critères de recherche

Le service recherche les séjours disponibles selon les critères suivants :

  1. Critères temporels : Le système identifie toutes les dates de début de séjour qui correspondent au jour de la semaine, mois et année spécifiés
  2. Filtrage par type de produit :
    • camping : Seuls les emplacements (tarifs de type 1)
    • location : Seuls les hébergements locatifs (tarifs de type 2)
    • Non spécifié : Tous types confondus
  3. Filtrage géographique : Par site(s) et/ou produit(s) spécifique(s)

Calcul des durées

  • Les durées sont calculées en jours entre la date de début et la date de fin de chaque séjour disponible
  • Seuls les séjours actifs (non dépréciés) sont pris en compte
  • Les durées sont regroupées et dédoublonnées

Cas spécifiques groupe

Pour les utilisateurs ayant des droits spécifiques groupe, le service retourne les identifiants externes des catégories plutôt que les identifiants internes des produits.

Exemples d'utilisation

Exemple de requête SOAP

<soap:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:web="http://webservices.webcamp.fr">
 <soapenv:Header/>
 <soap:Body>
   <web:cpg_StayDuration>
     <RequestCpgStayDuration_1>
       <idUser>USER_001</idUser>
       <password>motdepasse</password>
       <month>7</month>
       <dayOfWeek>7</dayOfWeek>
       <productType>location</productType>
       <searchPlace>
          <idProduct>
             <id1>321</id1>
             <idProduct>10</idProduct>
          </idProduct>
          <idSite>1111</idSite>
         </searchPlace>
         <year>2025</year>
         <idsite>1111</idsite>
     </RequestCpgStayDuration_1>
   </cpg_StayDuration>
 </soap:Body>
</soap:Envelope>

Exemple de réponse SOAP

<env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/">
 <env:Header></env:Header>
 <env:Body>
   <web:cpg_StayDurationResponse xmlns:web="http://webservices.webcamp.fr">
     <result xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
       <error>0</error>
       <idUser>USER_001</idUser>
       <stayDuration>
         <durationLength>7</durationLength>
         <productIds>
           <id1>321</id1>
           <idProduct>10</idProduct>
         </productIds>
       </stayDuration>
       <stayDuration>
         <durationLength>14</durationLength>
         <productIds>
           <id1>321</id1>
           <idProduct>10</idProduct>
         </productIds>
       </stayDuration>
     </result>
   </web:cpg_StayDurationResponse>
 </env:Body>
</env:Envelope>

Performances et limitations

  • Le service utilise des tables temporaires en mémoire pour optimiser les performances
  • Les données sont issues du cache des séjours (STAY_CACHE) pour une réponse rapide
  • Limitation à un maximum de résultats raisonnables pour éviter les timeouts

Authentification

L'authentification se fait via les paramètres idUser et password. Les droits d'accès déterminent les fonctionnalités disponibles (ex: accès aux identifiants externes pour les spécifiques groupe).
