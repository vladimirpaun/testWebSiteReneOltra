idOrderstringyes  L'identifiant de la réservation à modifier.
customerCustomerOrder    
changeTypestring  1 or more

Type de modification. A renseigner autant de fois que nécéssaire pour indiquer la nature du changement :
friends : pour indiquer une modification de participants
rubrics : pour indiquer une modification de rubriques de suppléments / Composition de séjour
stay : pour indiquer une modification du séjour (date, catégorie, tarif...)
comment : pour indiquer une modification de commentaires
orderTransfert : pour modifier le client réservant, ses accompagnants et ses options tout en conservant la disponibilité

supplementSupplementFilled  0 or moreNouvelle liste des suppléments
cancelEnableboolean   Définit s'il est possible de prendre l'option "assurance annulation" sur ce séjour.
cancelInsurancefloat   Montant TTC de l'option "assurance annulation" si cancelEnable = true.
pricefloat   Montant TTC du séjour.
commentstring   Commentaire.
payAmountfloat   Montant payé si le paiement a été effectué et si l'utilisateur a l'autorisation payChangeAmount. Sinon 0
idPaymentTypeIdPaymentType   Le type de paiement utilisé si le champ payAmount est différent de 0. Non utilisé sinon
paymentPaymentFormType   Les informations de paiement si le champ payAmount est différent de 0. Non utilisé sinon
